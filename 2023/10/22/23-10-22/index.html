动态规划

1.Longest Run on a Snowboard - UVA 10285 - Virtual Judge (vjudge.net)

  • 这和dfs有什么区别呢？
  • 有个特点，就是一次dp，可能就会得到很多点的最终状态；一个点在没有得到最终状态之前不会被询问到它的最终状态的，这得益于始终是H高的点去访问H低的点
  • 每个点向其四个方向去试探，直到不能延伸为止

2.Free Candies - UVA 10118 - Virtual Judge (vjudge.net)

  • 参数d指的是当前高度状态所对应的值
  • 每一层一般都有4种决策，只要每一堆还有糖果可拿
    • 如果框子里面有这个糖果，那么答案里就会拿走一对
    • 没有这个糖果，进行下一层的尝试

3.Cake slicing - UVA 1629 - Virtual Judge (vjudge.net)

  • Chery数组存储着樱桃所在的矩形的左上角位置，以及矩形的宽和高
  • 为了得到左上角坐标为r,c,宽和高度都确定的矩形内的Chery有多少个
  • 如果宽度大于1，那么就分成左右两块分开讨论
  • 如果宽度等于1，那么就分成上下两块进行讨论
  • 得到将每一块矩形切割为若干块每一块都只有一个Chery需要割多少刀
  • 对于每一层需要遍历横着切以及竖着切的每一种情况

4.Folding - UVA 1630 - Virtual Judge (vjudge.net)

  • 得到[l,r]内最小的循环单元的长度
    • 区间长度必能被循环单元的长度整除
  • 对于每一层，遍历[l,i]+[i+1,r],而且每一层的答案一个是自己处理了一遍，一个是直接从底层拼接

5.Stamps and Envelope Size

  • 题目中将会给出很多组的邮票数量和邮票面值，让你选出一个最佳
  • 如何选出一个最佳呢？首先可覆盖面值是最大的，其次邮票的数量是较少的，其次邮票的字典序排序是较小的
  • 如何得到一组邮票可以覆盖的最大面值呢？
    • 一个大的面额需要几张邮票呢？可能比较小的面额要多上几张邮票
    • 我们可以将其想象成从一个状态到另一种状态
      • 每一个状态代表满足这个面额至少需要几张邮票，而下一个状态代表这个状态加上1张邮票可以到达的面额
      • 但是要求面额必须是连续的，我们可以让当前的状态的面额减去一张邮票去找上一个状态，然后得到这个状态需要多少张邮票
    • 如果所需邮票数量过多，可以结束

6.Cyborg Genes - UVA 10723 - Virtual Judge (vjudge.net)

  • 这种处理方法比较神奇，可以避免空串的情况，也可以让len值成为S字符串的下标
  • T必须用这种方法来处理，因为会处理掉多余的换行符
  • 求两个字串的最短公共父串
